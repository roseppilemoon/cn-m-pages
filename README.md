# cn-m-pages

一个用于归档和发布独立 HTML 页面的公共仓库。

## 简介

本仓库主要用于存放和托管多个独立的 HTML 页面。这些页面可能包含信息展示、工具页面、静态内容等。仓库本身不针对特定域名或网站，仅作为页面存档与发布的中转站。

## 目录结构

```
cn-m-pages/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 公共资源文件（如样式、脚本、图片等）
├── README.md       # 本文件
└── .gitignore      # Git 忽略规则
```

- **pages/**：每个 HTML 页面为一个独立文件，命名清晰，便于识别。
- **assets/**：存放页面共用的静态资源，按需组织。

## 页面归档说明

- 所有页面均为静态 HTML，不依赖后端服务。
- 页面内容保持简洁，避免冗余代码。
- 页面之间相互独立，无跨页面依赖。
- 归档时建议在 commit 信息中简要说明页面用途或更新内容。

## 维护说明

- 欢迎提交新页面或改进现有页面，请遵循以下原则：
  - 页面文件名使用英文小写字母、数字、连字符，避免特殊字符。
  - 确保 HTML 代码结构完整、可读。
  - 避免在页面中嵌入外部不可控资源（如第三方脚本）。
- 本仓库不承担页面内容的长期可用性保证，建议维护者自行备份重要数据。
- 仓库维护者会定期清理无效或过时的页面。

## 使用方式

可直接通过 GitHub Pages 或其他静态托管服务访问 pages/ 目录下的 HTML 文件。仓库本身不提供导航或索引页面。

## 许可

本仓库内容无特定许可声明，贡献者默认同意其内容以仓库现有方式公开。如有特殊要求，请在提交时注明。
